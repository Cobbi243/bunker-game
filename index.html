Що зробити:

  1. Зайди на console.firebase.google.com
  2. Натисни "Add project" → дай назву → вимкни Analytics → створи
  3. На головній проекту: іконка </> (Web app) → дай назву → зареєструй
  4. Скопіюй об'єкт firebaseConfig
  5. У лівому меню: Build → Realtime Database → Create Database
  6. Регіон Europe → Start in test mode → Done
  7. Відкрий файл firebase-config.js у редакторі (поряд з index.html)
  8. Постав свої значення замість YOUR_API_KEY тощо
  9. Збережи та оновлюй сторінку

⚠ Файл firebase-config.js має лежати поряд із index.html. Якщо його немає — створи його за зразком (там вгорі коментар з прикладом).

🏚️ ЗАГАЛЬНА КОНЦЕПЦІЯ

Стається апокаліпсис. Єдиний шанс вижити — потрапити до бункера, але місць у ньому мало. Кожен гравець отримує випадкові характеристики персонажа і повинен переконати інших, що саме він заслуговує вижити.

🎤 ФАЗА 1: ПРЕЗЕНТАЦІЯ

Гравці по черзі виступають. На своєму ході — натискаєш кнопку «РОЗКРИТИ» біля характеристики, яку хочеш показати всім.

⚠ ОБОВ'ЯЗКОВЕ РОЗКРИТТЯ

1-й раунд: розкрити 2 характеристики

Кожен наступний раунд: по 1 новій

💬 ФАЗА 2: ОБГОВОРЕННЯ

Дискусія, питання, союзи. Тут можна використовувати карти подій. Ведучий може кинути катаклізм.

🗳 ФАЗА 3: ГОЛОСУВАННЯ

Кожен голосує за одного гравця. Найбільше голосів — виключається. Картка виключеного розкривається повністю.

🎴 КАРТА ПОДІЇ (1 НА ГРУ)

На початку гри кожен отримує 1 випадкову карту події. Використовуй коли хочеш — за всю гру лише один раз. Можливі карти:

  • Розкриття — змусь іншого показати одну характеристику
  • Подвійний голос — твій голос = 2
  • Захист — врятуй когось (або себе) від виключення
  • Обмін — заміни свою характеристику на нову випадкову
  • Шпигунство — таємно подивись чужу характеристику (бачиш тільки ти)
  • Мовчання — обраний гравець не голосує в наступному раунді
  • Анонімний голос — твій голос не відображається у статистиці
  • Перенаправлення — голос проти тебе перенаправь на іншого
  • Приховування — сховай назад одну раніше розкриту характеристику
  • Секрет назовні — розкрий секрет (8-му характеристику) обраного гравця
  • Додаткова карта — отримай ще одну випадкову карту
  • Поміняти подію — обміняй свою карту на нову випадкову
  • Медогляд — таємно подивись здоров'я всіх живих гравців
  • Вампір — якщо тебе виключать, забери з собою одного гравця
⚡ КАТАКЛІЗМ (1 НА ГРУ)

На випадковому раунді (1-4) автоматично спрацьовує один катаклізм:

  • Епідемія — найхворіший вибуває
  • Прорив — місткість бункера −1
  • Викриття — всі секрети розкриваються
  • Ворожнеча — двоє не можуть голосувати один проти одного
  • Подвійне виключення — виключаються двоє
  • Амнезія — гравець забуває розкриття
  • Розширення — місткість +1

v1.9 — Автоматична загроза + детальні бункери
  • ⏳ Автоматична загроза — після закриття бункера є 3-секундна драматична пауза, а потім карта загрози випадає сама. Хосту більше не треба нічого натискати.
  • 🏚 Детальні описи бункерів — тепер кожен бункер має конкретну площу, ресурси на скільки років, кількість запасів, особливі переваги і недоліки. Стратегічне планування набагато цікавіше.
  • Узгодженість опису — конкретна кількість людей у бункері визначається тільки хостом (через налаштування «Місцьь у бункері»). В описах залишилась тільки площа та ресурси, без точних цифр людей.
v1.8 — Карти загроз + логічні характеристики
